# Thread, Process, Hardware, OS, and AI

## Overview

This section introduces core concepts in **computer architecture** and **operating systems** that are essential for understanding how software runs efficiently on modern hardware.

It covers:

- CPU internals and execution flow
- Memory hierarchy and isolation
- Threading and multiprocessing
- Process and thread behavior in **GPOS** and **RTOS**
- Scheduling strategies and OS design
- Hardware-aware computing concepts such as **NUMA**
- Evolving processor models for AI-oriented workloads

## Learning Goals

By working through these notes, you should be able to:

- Understand how CPU components cooperate during execution
- Compare memory types and their roles in system performance
- Distinguish between **threads** and **processes**
- Explain the difference between **multithreading** and **multiprocessing**
- Understand scheduler behavior in different operating system models
- Recognize how hardware design impacts software performance
- Build a stronger foundation for systems programming and AI infrastructure

## Why This Section Matters

A solid understanding of hardware and OS fundamentals helps you:

- Write more efficient programs
- Debug performance bottlenecks more effectively
- Make better architectural decisions
- Understand concurrency, scheduling, and resource sharing
- Prepare for advanced topics in systems, distributed computing, and AI engineering
